Simulation Control and Reporting — Curated Overview

This local page is a curated helper for Ray-Modeler. For authoritative and complete documentation, refer to the official EnergyPlus Input Output Reference for your version.

SimulationControl

SimulationControl defines which parts of the simulation are run (sizing periods, weather run periods), and some high-level options.

  • Do Zone Sizing Calculation — Yes/No for zone sizing.
  • Do System Sizing Calculation — Yes/No for system sizing.
  • Do Plant Sizing Calculation — Yes/No for plant sizing.
  • Run Simulation for Sizing Periods — Run design day simulations.
  • Run Simulation for Weather File Run Periods — Run actual run periods.

In Ray-Modeler, high-level "run" settings should map to these flags. Ensure that at least one of sizing periods or run periods is enabled.

RunPeriod

RunPeriod specifies the start/end dates (and days) for a simulation run. Multiple run periods may be defined; names can be referenced in other objects.

  • Start and end month/day define the time span.
  • Day of week and holiday behavior control calendar alignment.
  • Adjustments exist for leap years and holidays.

When Ray-Modeler exposes "simulation period" or "analysis period" settings, they should be translated into one or more RunPeriod objects.

Timestep

Timestep sets the number of simulation timesteps per hour. Common values are 4 (15 min) or 6 (10 min).

A smaller timestep (larger number) increases temporal resolution and runtime. Ray-Modeler run settings should clearly indicate the chosen timestep.

Reporting Basics

EnergyPlus supports several reporting objects. Key ones typically relevant to Ray-Modeler workflows:

  • Output:Variable — Request specific variables (e.g. zone loads, surface temperatures) at a chosen frequency.
  • Output:Meter — Request energy and demand from predefined meters.
  • Output:Table:SummaryReports — Enable standard summary tables.

Ray-Modeler may generate a minimal set of reporting objects automatically to support its dashboards and post-processing. Users can reference the official I/O Reference for detailed lists of variable and meter names.
