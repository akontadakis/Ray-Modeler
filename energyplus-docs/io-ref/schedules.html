Schedules — Curated Overview

This page explains how Ray-Modeler schedules correspond to EnergyPlus schedule objects. It focuses on ScheduleTypeLimits and Schedule:Compact, which are commonly used for loads, thermostats, availability, and controls.

ScheduleTypeLimits

ScheduleTypeLimits defines the valid range and type for schedules: temperature, fraction, on/off, etc.

  • Lower/Upper Limit Value — Allowed numeric range.
  • Numeric Type — CONTINUOUS or DISCRETE.
  • Unit Type — e.g. Temperature, Fraction, Dimensionless.

Ray-Modeler can assume standard limits for common schedule types (e.g. 0–1 for Fraction). Explicit mapping is handled in the model builder based on schedule usage.

Schedule:Compact

Schedule:Compact defines time-varying values using a readable, rule-based syntax (Through/For/Until).

  • Lines are evaluated in order: Through → For → Until.
  • Values can represent fractions, temperatures, etc., depending on TypeLimits.
  • Used widely for: people, lights, equipment, infiltration, setpoints, IdealLoads availability, etc.

Ray-Modeler’s Schedules panel writes user-defined schedules as Schedule:Compact blocks. Built-in schedules (e.g. RM_AlwaysOn) are provided as predefined compact schedules.

Mapping summary: - Ray-Modeler custom schedules → Schedule:Compact with appropriate ScheduleTypeLimits. - Many configuration panels reference schedule names; those names must match the schedules defined here.
