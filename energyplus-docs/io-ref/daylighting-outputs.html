Daylighting and Outputs — Curated Overview

This page links the Ray-Modeler "Daylighting & Outputs" configuration to key EnergyPlus objects. For full details, see the official I/O Reference.

Daylighting:Controls

Daylighting:Controls defines one or two reference points in a zone, the desired illuminance setpoint, and how lights are controlled.

  • Zone Name — Target zone.
  • Reference Point(s) — Coordinates (x,y,z).
  • Illuminance Setpoint — Target [lux].
  • Lighting Control Type — Continuous, Stepped, etc.
  • Lighting Control Fraction — Portion of lights controlled.

Ray-Modeler per-zone daylighting settings map directly to Daylighting:Controls with the configured reference points, setpoint, control type, and fraction.

Output:IlluminanceMap

Output:IlluminanceMap requests grid-based illuminance results on a plane within a zone.

  • Zone Name
  • Origin (x,y,z) — Lower-left corner.
  • Number of Points and Spacing — Nx, Dx, Ny, Dy.

Ray-Modeler illuminance map definitions map to Output:IlluminanceMap entries using the specified grid and zone.

Output:Variable (Basics)

Output:Variable requests reporting of specific simulation variables at a chosen frequency.

  • Key Value — e.g. zone name, "Environment", or "*".
  • Variable Name — exact name of the variable.
  • Reporting Frequency — Timestep, Hourly, RunPeriod, etc.

Ray-Modeler Output Variables table maps directly to Output:Variable objects. Use these to enable metrics required by your workflows.

Mapping summary: - Daylighting controls UI → Daylighting:Controls. - Illuminance maps UI → Output:IlluminanceMap. - Output variables UI → Output:Variable entries for selected keys/variables/frequencies.
