Zone Loads, Thermostats, and IdealLoads — Curated Overview

This page links the Ray-Modeler "Zone Loads / Thermostats / IdealLoads" configuration to key EnergyPlus input objects. For full details, see the official I/O Reference.

People

People describes occupants and their internal gains.

  • Number of People / People per Zone Floor Area
  • Fraction Radiant / Latent / Sensible
  • Activity Level Schedule
  • People Schedule Name — occupancy schedule.

Ray-Modeler "People [p/m²]" with a schedule maps to a People object using the specified density and schedule.

Lights

Lights defines lighting power and schedules.

  • Watts per Zone Floor Area
  • Schedule Name — lighting operation.

Ray-Modeler "Lights [W/m²]" + schedule → Lights with Watts/Area and Schedule:Compact.

ElectricEquipment

ElectricEquipment represents plug/process loads.

  • Watts per Zone Floor Area
  • Schedule Name

Ray-Modeler "Equip [W/m²]" + schedule → ElectricEquipment.

ZoneInfiltration:* (e.g. DesignFlowRate)

Infiltration objects describe uncontrolled air leakage.

  • Common form: ZoneInfiltration:DesignFlowRate.
  • Rates can be given in ACH, m³/s, etc.
  • Schedule controls when infiltration applies.

Ray-Modeler "Infil [ACH]" + schedule → a ZoneInfiltration:DesignFlowRate using the ACH method for the target zone.

DesignSpecification:OutdoorAir

DesignSpecification:OutdoorAir defines per-person and per-area ventilation.

  • Methods: Flow/Person, Flow/Area, or combinations.
  • Referenced by zone equipment (including IdealLoads).

Ray-Modeler IdealLoads OA settings (L/s.person, L/s.m²) can be expressed using DesignSpecification:OutdoorAir and referenced from IdealLoads.

ZoneControl:Thermostat

ZoneControl:Thermostat links zones to heating and cooling setpoint schedules.

  • One or more setpoint schedules (single, dual, etc.).
  • Referenced by HVAC/IdealLoads serving the zone.

Ray-Modeler thermostat configuration (heating / cooling schedule names) → ZoneControl:Thermostat entries with those schedules.

ZoneHVAC:IdealLoadsAirSystem

ZoneHVAC:IdealLoadsAirSystem simulates an idealized zone unit to meet loads.

  • Availability Schedule
  • Heating/Cooling Limit Type and capacities.
  • Outdoor Air: design OA method and flows (often via DesignSpecification:OutdoorAir).

Ray-Modeler "IdealLoads" settings (availability, caps, OA) populate ZoneHVAC:IdealLoadsAirSystem fields per zone or globally.

Mapping summary: - Zone Loads panel → People, Lights, ElectricEquipment, ZoneInfiltration:* with schedules. - Thermostats → ZoneControl:Thermostat with heating/cooling setpoint schedules. - IdealLoads → ZoneHVAC:IdealLoadsAirSystem (+ DesignSpecification:OutdoorAir) for capacities and OA.
